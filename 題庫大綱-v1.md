# 專案起始新手包 — 題庫大綱 v1

> 用途：給非技術 PM 的「全域規範問答精靈」題庫。
> 本檔與 `index.html` 的 `CONFIG` 同步；改題庫時兩邊一起更新。
>
> 標記：🔧 技術/合約層（關卡五，可整關隱藏）｜【新增】補的缺口項｜【推薦】預設建議選項｜↳ 分支邏輯｜💡 為什麼問/差異說明
>
> 架構原則：PM 能決策的（關卡一～四）在前，技術/合約（關卡五）收最後；關閉「技術題 🔧」開關即隱藏整個關卡五。

## 關卡一　系統定位
> 系統是什麼、給誰用、跑在哪些裝置與語言環境。

### 1-1　基本設定
- **專案系統類型**：業務管理後台 / 民眾網前台 / 獨立儀表板 / Kiosk
  - ↳ 選「民眾網前台」→ 解鎖關卡二的 SEO、Cookie 題
- **適用裝置 (RWD)**：電腦尺寸即可 / 全尺寸響應式 / 特定尺寸（↳ 自填尺寸）
  - ↳ 選「電腦尺寸即可」→ **最低支援視窗寬度**：1024 / 1280 / 1366 / 1440 / 1920 px / 其他
    - 💡 1366px 最常見筆電解析度、1920px 為 Full HD
- 【新增】**支援瀏覽器清單**（複選）：Chrome【推薦】/ Edge【推薦】/ Safari / Firefox / IE 相容
- **是否需要無障礙**：不需要 / 是
  - ↳ 選「是」→ **等級**：A / AA【推薦】/ AAA
    - 💡 三級累加（AA 含 A、AAA 含 AA）。A=基本門檻（替代文字、可鍵盤操作、不單靠顏色）；AA=好用（對比 4.5:1、放大 200% 不破版、focus 可見）為政府/法規標準；AAA=最嚴（對比 7:1、手語影片、無時限），整站難全達成
- **是否需要 GA 分析**：不需要 / 是（↳ 追蹤碼 G-XXXXXXXX）

### 1-2　語言與在地化
- **是否需要多語系**：不需要 / 是
  - 💡 多語系獨立一個需求編號；翻譯由誰做、翻譯社、實作方式留訪談寫細節
  - ↳ 選「是」→ **支援語言**（複選）：繁體中文【推薦】/ 英文【推薦】/ 日文 / 韓文；另有「其他語言」自填
- 【新增】**在地化格式**（日期／時區／金額）：自填

## 關卡三　帳號與登入
> 登入、密碼、權限與 Token／Session 行為。

### 3-1　登入設定
- **是否需要驗證碼**：是 / 否（↳ 樣式：6 碼數字 / 6 碼英數 / 6 碼英文 / reCAPTCHA / 其他）
- **是否需要忘記密碼功能**：是 / 否
- 【新增】**是否需要雙因素驗證（2FA／OTP）**：不需要 / 需要（↳ 方式，例 Email/簡訊 OTP、Authenticator）
- **初次登入是否強制變更密碼**：是 / 否

### 3-2　錯誤鎖定
- **密碼錯誤是否需鎖定**：是（↳ 次數上限／鎖定時間）/ 否
  - ↳ 選「是」才解鎖下方鎖定模式
- **鎖定模式**：鎖定帳號【推薦】/ 鎖定當前瀏覽器
  - 💡 鎖定帳號＝DB 記錄，換瀏覽器/重整/換裝置都無法解除（注意可能被惡意鎖死）；鎖定瀏覽器＝重啟即重置，較寬鬆但防護弱。政府/高安多選前者

### 3-3　密碼規範與效期
- **密碼規範**（複選）：長度 8–20 碼 / 含數字 / 含英文大寫 / 含英文小寫 / 含特殊符號 `#?!@$%^&*-`
- **輸入字元限制**：不可輸入＿＿ / 僅限輸入＿＿（無則留空）
- **密碼最長效期**：不需要 / 有（↳ 30 / 60 / 90 天 / 其他）
- **密碼最短效期**：不需要 / 有（↳ 1 / 2 / 3 天 / 其他）
- **變更密碼不可與前幾次相同**：不需要 / 有（↳ 3 / 5 / 6 次 / 其他）

### 3-4　權限設定
- 可否變更他人密碼：是 / 否
- 可否變更自己密碼：是 / 否
- 可否刪除／停用自己帳號：是 / 否
- 可否刪除／停用自己帳號的權限角色：是 / 否

### 3-5　Token / Session
- **Token 效期**：1hr / 8hr / 24hr / 7 天 / 其他
- **Token 更新**：是 / 否
- **關閉瀏覽器是否保持登入**：是 / 否
- **同瀏覽器允許分頁登入不同帳號**：是 / 否
- 【新增】**同一帳號是否限制單一登入**：是（新登入踢掉舊的）/ 否
- **Token 失效檢查方案**（複選）：一段時間主動檢查 / 動作檢查 / 進入頁面檢查
  - 💡 主動檢查＝最即時但耗資源；動作檢查＝省資源但閒置不察覺；進入頁面＝最省但停同頁不察覺。可併用（常見：動作＋進入頁面）
- **自動登出倒數重置行為**（複選）：click【推薦】/ contextmenu【推薦】/ keydown【推薦】/ mousemove / scroll
  - 💡 mousemove、scroll 觸發極頻繁，可能效能問題；建議只開前三

## 關卡四　資料與通知
> 通知寄送、資料治理與檔案處理等功能性全域決策（多為原清單常漏項）。

### 4-1　通知寄送
- 【新增】**是否需要系統通知**：需要 / 不需要
  - ↳ 選「需要」才展開以下
- 【新增】**通知管道**（複選）：Email / 簡訊 / 站內訊息
- 【新增】**寄信方式**：客戶 SMTP / 第三方（SendGrid 等）/ 待定（↳ 通知管道含 Email 才出現）
- 【新增】**觸發情境**：自填（例：註冊、忘記密碼、審核結果、到期提醒）

### 4-2　資料治理
- 【新增】**操作稽核軌跡 (audit log)**：需要【推薦】（↳ 記錄哪些動作）/ 不需要
  - 💡 政府案幾乎都要求操作軌跡
- 【新增】**資料保存期限 / 自動清除**：有（↳ 期限）/ 無
- 【新增】**資料匯出**：需要（↳ 格式 Excel/CSV/PDF）/ 不需要
- 【新增】**個資 / 刪除權**：有個資（↳ 去識別／刪除機制）/ 無

### 4-3　檔案處理
- 【新增】**檔案上傳限制**：大小上限＿＿、格式＿＿
  - 💡 建議一併確認是否需要上傳檔案病毒掃描

## 關卡五　技術、資安與部署　🔧
> 技術選型、資安檢測與機房部署，多由工程／合約決定。關閉「技術題 🔧」開關可整關隱藏。

### 5-1　資安檢測報告
- **原始碼檢測**：不需要 / 是（↳ 頻率／檢測方／工具，例 SonarQube）
- **弱點掃描**：不需要 / 是（↳ 頻率／工具／等級，例 Acunetix、不得有中風險度以上）
- **壓力測試**：不需要 / 是（↳ 頻率／工具，例 JMeter）
  - 💡 詳細壓測需求另出文件給 QA
- **滲透測試**：不需要 / 是（↳ 頻率／檢測方）
- **資通系統防護基準檢核表**：不需要 / 是（↳ 等級 高/中/普）
- **客戶自有資安表單**：不需要 / 是（↳ 表單名稱）

### 5-2　連線與機房
- **正式機**：架設環境（客戶/鎰威/其他）｜連線（外網/內網）｜SSL（是/否）
- **客戶測試機**：沒有測試機 / 架設環境（↳ 連線，選「沒有」則跳過）
- **鎰威測試機**：架設環境｜連線
- **備份機制**：是 / 否　│　**備援機制**：是 / 否
  - 💡 備份＝救資料（可還原但服務會中斷）；備援＝救服務（自動切換不中斷），兩者互補

### 5-3　開發環境
- **前端**：Vue 3 / 其他
- **後端**：.NET Core / 其他
- **資料庫**：MSSQL / MySQL / 其他
- **UI 庫**：Ewill Guideline–Vuetify / 客戶或合作廠商自訂（例 Shadcn）
- **部署方式**：VM / Docker / K8s / 其他
- **版控**：GitLab / 其他
